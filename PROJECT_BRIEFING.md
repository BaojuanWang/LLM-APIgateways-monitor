## 一、这是什么研究

一篇**测量 / 安全研究论文**,刻画 **LLM API 中转站("中转站"/relay)生态**——
就是那些第三方网关,低价转售 OpenAI/Claude/Gemini 等商业大模型 API 的站点。

**研究视角:供给侧基础设施刻画**——它们用什么技术栈、部署在哪、谁在运营、
怎么变现、生态多集中。
**明确不做**:数据面滥用(模型掺水/身份不符那类),那是另一篇论文的地盘,不碰。

一句话:**我们在给这个灰色生态"拍 X 光片",看它的骨架结构。**

---

## 二、系统怎么搭的(两层架构)

1. **发现层(discovery)**——用两种独立方法找站点:
   - **GitHub 代码搜索**:靠开源框架(one-api 等)的代码指纹去找 → 找到 764 个
   - **FOFA 网络空间测绘**:不靠框架、靠网络特征去找 → 又找到 280 个新的
   - 合起来去重后 **1089 个站**(按主域归并,一个站一行)

2. **分析层(analysis)**——对找到的站做分组、分类、画像。**纯本地计算,不重新采集**,
   换新数据重跑一遍就出新结果。这是这个窗口这两天主要在做的部分。

数据采集分两种环境:
- **能在云端沙箱做的**:纯本地计算(分类/交叉表/网络图)。
- **必须在你本地机器做的**:凡是要连外网的(证书、WHOIS、隐私政策爬取)——
  因为云端沙箱的网络被限制。所以你会看到我经常说"这个你本地跑一下"。

## 三、已经得出的核心结论(论文能用的)

### 结论 1:生态是"技术栈单一化"的(近乎单一栈)
one-api 这个开源框架家族占了压倒性多数。**风险点**:全生态用同一套软件 =
一个漏洞可能打穿一大片(软件单一化风险,学术上叫 monoculture)。

### 结论 2:发现方法本身有偏差(这是最硬的一块论证,§4.1)
- 用 **GitHub 代码搜索**找到的站:**96%** 是 one-api —— 但这是"循环论证",
  你靠框架指纹找,当然找到的都有框架。
- 换 **FOFA**(不靠框架)找到的站:one-api 只占 **71%**,还冒出 **24% 的异构尾巴**
  (GitHub 那种方法根本看不见的、五花八门的自建站)。
- **深挖那 24% 尾巴**(逐站探测)后:其中只有 8% 是伪装的 one-api,89% 是**真异构**。
  把漏检的加回来,集中度从 71% 收紧到 **73%**。

**这三个数字(96% / 71% / 73%)一起说明:**
① 集中是真的(不是找法造成的假象);
② 但代码搜索**高估**了集中度;
③ 代码搜索**系统性漏掉**了一大片真实存在的异构小站。
→ 完整论证见 `docs/FINDING_discovery_bias.md`。

### 结论 3:运营者其实很分散(集中在"技术"不在"人")
1089 个站 → 归并出 **1000 个运营者**,最大的一家也只控 6 个站。
所以"集中"是指**大家都用同一套软件**,不是**少数人垄断**。这个区分很重要。

### 结论 4:生态极年轻、在井喷
六成站是 2026 年才注册的,2026 上半年逐月新增(5月峰值 181 个)。

### 结论 5:傍品牌 / 商标风险
41 个站直接把 openai/claude/gpt 等品牌名塞进域名,其中 33 个还活着在营。

## 七、几个术语速查(看报告会遇到)

- **one-api / new-api**:最流行的开源中转站框架名。"one-api 家族"= 这一系列。
- **eTLD+1 / 主域归并**:把 `api.x.com`、`www.x.com` 都算作同一个站 `x.com`。
- **运营者归并(operator matching)**:靠共享证书/favicon/IP/站名,判断多个域名是不是同一伙人。
- **模板家族(template family)**:靠共享搭建特征,判断是不是同一个"搭建方"批量搭的(未必同一人)。
- **HHI**:集中度指数。这里很低(≈0.0015)= 运营者分散。
- **保守下界 / 收紧下界**:71% 是集中度的最低估计,深挖后收紧到 73%。
- **CDN fronted**:藏在 Cloudflare 等 CDN 后面,看不到真实源站 = 刻意不透明。
- **faka / 发卡**:自动化售卡变现(发卡网)。
