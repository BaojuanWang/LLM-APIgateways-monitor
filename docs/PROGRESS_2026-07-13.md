## 背景
1089 个 LLM 中转站,目标:追运营者真实身份 + 量化"追不到"的结构性机制。
今天补齐三块数据并分析。

---

## ① 隐私政策爬取(全量跑完)
- **16%(182/1089)有隐私政策** —— 但这是**静态爬虫下界**:采集器是 `requests` 静态抓取,
  抓不到 React/Vue 站(占 ~50%)JS 渲染后的内容,真实值更高(待 Playwright 重抓验证)。
- 有政策的站里,**88% 连适用法律都不声明**;只有 4% 中国法 / 3% 新加坡 / 2% GDPR。
  → 隐私政策多为形式化空文。
- 佐证爬虫局限的实测:支付方式覆盖率 **JS 站 9% vs 非 JS 站 25%(差 2.8 倍)** —— 低覆盖
  主要是静态爬虫抓不到 JS 内容,不是站没有。

## ② 注册人身份(WHOIS 迁移)—— 核心结论:可追溯率
- 给全部站重查 WHOIS 补注册人字段:34% 返回注册人机构,但**过滤隐私代理占位后只剩 38 个真实体**。
- **合并硬信号(WHOIS 真实注册人 38 + 证书 Org 6)→ 可追溯率 = 4.0%(44/1089)**。
- **三层反追溯机制(带覆盖率)**:
  - 网络层:70% 藏 CDN 后,源站 IP 不可见
  - 注册层:31% WHOIS 用隐私代理占位(Domains By Proxy / Withheld for Privacy / DATA REDACTED …)
  - 证书层:~99% 免费 DV 证书,无企业实名字段
- 结论:**96% 的站结构性无法归因到真实运营者** —— 把"查不到"翻转成量化 finding(不依赖软推断)。
- 漏出真实实体的 44 个站是高价值线索(海南斯拓智能科技、ZHENBOYUAN INDUSTRIAL、蒋仁保、王鹏 …)。

## ③ Reverse-IP 找马甲(抽样 21/287 IP)
- 从 21 个已反查 IP 挖出 **12 个"跨品牌同驻"候选马甲**,涉及 6 个运营者扩张:
  - anyroute.io + kitcode.ai / pomoai.ai / pomoai.xyz / qweapi.com
  - 4sapi.com + 4sapi.club / 4sapi.org;aigptx.top + opapi.win / wisegpt.top
  - ainstant.pro + kwwaigc.cn / xinyuai.cc;aigc456.top + ai98.vip;ccfly.codes + lingxu.ai
- ⚠️ **诚实边界(重要)**:HackerTarget 是 passive DNS(历史被动 DNS),这些是**候选、非确认活马甲**。
  **人工点开验证:12 个里大部分打不开,只有 api.maynor1024.live 确认在营。** 故这批混着:
  真活马甲 / 已死(中转站 churn 高、跑路换壳)/ 历史短暂同驻噪声。
- **未决**:需加"死活 + 当前 IP"验证,只保留"活着且现在仍解析到同一 IP"的算确认马甲;
  已死的降级为"域名轮换"弱证据。

  别把 passive-DNS 毛数当确认值。
